Flash Loan Arbitrage on Solana

Zero-capital MEV extraction using Marginfi flash loans. Borrow, arbitrage, repay — all in a single atomic transaction.

April 2, 2026 8 min read Technical Guide

Flash loans are the great equalizer in MEV. They let you execute arbitrage without deploying any capital — borrow millions in SOL, execute a profitable trade, repay the loan, and keep the profit, all within a single Solana transaction. If the trade isn't profitable, the transaction reverts and you lose nothing but the transaction fee.

On Solana, Marginfi is the primary flash loan provider. This article covers how flash loans work, how to integrate them into arbitrage bots, and the economics that make this profitable.

How Flash Loans Work on Solana

A flash loan on Solana follows the borrow-use-repay pattern within a single transaction:

  1. Borrow: Your transaction calls the Marginfi flash loan instruction, borrowing tokens from the lending pool
  2. Use: Intermediate instructions in the same transaction use the borrowed tokens (swaps, arbitrage, liquidations)
  3. Repay: Before the transaction ends, you repay the borrowed amount plus a fee
  4. Verify: The Marginfi program verifies the repayment in a final instruction

If the repayment instruction fails or is missing, the entire transaction reverts — including the borrow. This atomicity is what makes flash loans risk-free for the lender and capital-free for the borrower.

Marginfi Flash Loan Mechanics

Marginfi's flash loan system works through its existing lending pool infrastructure. The available flash loan liquidity equals the idle liquidity in each lending pool. Key parameters:

  • Flash loan fee: Typically 0.01-0.05% of borrowed amount (varies by token and market conditions)
  • Available liquidity: Depends on pool utilization. SOL and USDC pools usually have the deepest liquidity
  • Transaction scope: Borrow and repay must occur within the same transaction
  • No collateral required: The atomic repayment guarantee eliminates collateral risk

Flash Loan Arbitrage Architecture

A typical flash loan arbitrage transaction has this instruction sequence:

Transaction Instructions:
1. Marginfi: flash_loan_begin(borrow 1000 SOL)
2. DEX A:    swap(1000 SOL → USDC)         // Buy USDC where it's cheap
3. DEX B:    swap(USDC → SOL)              // Sell USDC where it's expensive
4. Marginfi: flash_loan_end(repay 1000 SOL + fee)
5. Program:  verify_profit(min_profit = X)  // Revert if not profitable

The custom verify program (instruction 5) is critical — it checks that after all swaps and loan repayment, your token account has more than it started with. If the profit is below your minimum threshold, the entire transaction reverts.

Implementation Guide

Step 1: Set Up Marginfi Account

Before using flash loans, you need a Marginfi user account. This is a one-time setup:

// Create Marginfi account (one-time)
let create_account_ix = marginfi::instruction::create_account(
    &marginfi_program_id,
    &marginfi_group,
    &payer.pubkey(),
)?;

Step 2: Build the Flash Loan Transaction

The transaction combines Marginfi flash loan instructions with your arbitrage swaps:

fn build_flash_loan_arb_tx(
    borrow_amount: u64,
    borrow_mint: &Pubkey,
    swap_a_ix: Instruction,    // Buy on cheap DEX
    swap_b_ix: Instruction,    // Sell on expensive DEX
    marginfi_group: &Pubkey,
    marginfi_account: &Pubkey,
    payer: &Keypair,
) -> Result<Transaction> {
    let flash_loan_begin = marginfi::instruction::flash_loan_begin(
        marginfi_group,
        marginfi_account,
        borrow_mint,
        borrow_amount,
    )?;

    let flash_loan_end = marginfi::instruction::flash_loan_end(
        marginfi_group,
        marginfi_account,
        borrow_mint,
        borrow_amount,  // Repay amount (fee added by program)
    )?;

    let verify_profit_ix = build_verify_profit_instruction(
        &payer.pubkey(),
        borrow_mint,
        MIN_PROFIT_LAMPORTS,
    )?;

    let instructions = vec![
        ComputeBudgetInstruction::set_compute_unit_limit(400_000),
        ComputeBudgetInstruction::set_compute_unit_price(priority_fee),
        flash_loan_begin,
        swap_a_ix,
        swap_b_ix,
        flash_loan_end,
        verify_profit_ix,
    ];

    Ok(Transaction::new_signed_with_payer(
        &instructions,
        Some(&payer.pubkey()),
        &[payer],
        recent_blockhash,
    ))
}

Step 3: Optimal Borrow Amount

The profit from an arbitrage is not linear with size. Larger trades have more slippage, reducing the effective price difference. The optimal borrow amount maximizes profit - flash_loan_fee - tx_fees - jito_tip.

Find the optimum using binary search:

fn find_optimal_borrow(
    pool_a: &PoolState,
    pool_b: &PoolState,
    flash_loan_fee_bps: u64,
) -> u64 {
    let mut lo = MIN_BORROW;
    let mut hi = MAX_BORROW;  // Limited by pool liquidity

    while hi - lo > PRECISION {
        let mid = (lo + hi) / 2;
        let profit_mid = simulate_arb_profit(pool_a, pool_b, mid);
        let profit_next = simulate_arb_profit(pool_a, pool_b, mid + STEP);

        if profit_next > profit_mid {
            lo = mid;
        } else {
            hi = mid;
        }
    }

    (lo + hi) / 2
}

Economics of Flash Loan Arbitrage

Understanding the cost structure is critical for profitability:

Cost Breakdown

  • Flash loan fee: 0.01-0.05% of borrowed amount. On a 1000 SOL borrow at 0.03%, that's 0.3 SOL (~$45 at $150/SOL)
  • Jito tip: Varies by competition. Typically 0.001-0.05 SOL for mid-tier opportunities
  • Priority fee: Based on CU consumption. A 400K CU transaction at 100K microlamports/CU = 0.04 SOL
  • Base transaction fee: 0.000005 SOL — negligible
  • Swap fees: DEX trading fees (0.01-0.3% per swap depending on pool)

Profitability Threshold

For a flash loan arb to be profitable, the price discrepancy must cover all costs:

Required spread > flash_loan_fee + swap_fee_a + swap_fee_b + jito_tip + priority_fee

Example:
  Flash loan fee (0.03%):     0.300 SOL
  Swap fee DEX A (0.25%):     2.500 SOL
  Swap fee DEX B (0.01%):     0.100 SOL
  Jito tip:                   0.010 SOL
  Priority fee:               0.040 SOL
  ────────────────────────────────────
  Total cost:                 2.950 SOL

  With 1000 SOL position, need >0.295% price spread to break even

Why Flash Loans Still Win

Despite the additional flash loan fee, flash loans are often more profitable than self-funded arbitrage because:

  • Unlimited position sizing: Take maximum size on every opportunity instead of being limited by your balance
  • No capital risk: Your funds sit safely in your wallet. A bug in your strategy can't drain your capital because the flash loan reverts.
  • No rebalancing: Self-funded arbs accumulate inventory in various tokens. Flash loans always start and end in the same token.
  • Capital efficiency: Your capital can be deployed elsewhere (staking, lending) while flash loans handle arbitrage

Multi-Hop Flash Loan Arbitrage

Flash loans shine in multi-hop scenarios where the required capital for self-funding would be prohibitive:

Transaction Instructions:
1. Marginfi:   flash_loan_begin(borrow 5000 USDC)
2. Orca:       swap(5000 USDC → SOL)
3. Raydium:    swap(SOL → BONK)
4. PumpSwap:   swap(BONK → USDC)
5. Marginfi:   flash_loan_end(repay 5000 USDC + fee)
6. Program:    verify(USDC_balance > initial + min_profit)

This 3-hop path through SOL and BONK exploits price discrepancies across three different DEXes. Without flash loans, you'd need $750K+ in USDC to make this trade at scale.

Flash Loan Limitations

Flash loans are powerful but not unlimited:

  • Liquidity constraints: You can only borrow what's available in the lending pool. During high-utilization periods, available flash loan liquidity decreases.
  • CU budget: Complex multi-hop flash loan arbs consume significant compute units. Solana's 1.4M CU limit per transaction can be restrictive for 4+ hop paths.
  • Supported tokens: Marginfi only offers flash loans for tokens that have active lending pools (SOL, USDC, USDT, and select others).
  • Atomicity requirement: Everything must happen in one transaction. You can't split a flash loan strategy across multiple transactions.

Alternatives to Marginfi

While Marginfi is the primary flash loan provider on Solana, alternatives exist:

  • Kamino: Offers flash loan functionality through its lending pools with competitive fees
  • Solend: Original Solana lending protocol with flash loan support
  • Custom pools: Some teams build their own flash loan pools for exclusive access and lower fees

In production, it's best to support multiple flash loan providers and select based on available liquidity and fee at execution time.

Putting It Together

Flash loan arbitrage is the most capital-efficient MEV strategy on Solana. Combined with a fast data pipeline, smart routing engine, and multicast submission, it enables profitable extraction without significant capital deployment. The barrier to entry is technical — you need excellent Rust skills, deep protocol knowledge, and robust infrastructure — but the capital barrier is near zero.
